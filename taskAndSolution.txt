 Задание 21.6.3
 Создайте собственный плагин.

 1. Создаем новый проект в idea с системой сборки maven

	name:		MyPlugin
	groupId:	org.sf
        artifactId:	version-maven-plugin	<-- здесь должна быть именно эта строчка
        version:	1.123-SNAPSHOT
                                   
 2. Добавляем: 
	
	<encoding>UTF-8</encoding> в блок properties чтобы при сборке mvn не ругался на кодировку

 3. Устанавливаем версию Java 1.8 иначе сборка не походит(((

	<maven.compiler.source>8</maven.compiler.source>
        <maven.compiler.target>8</maven.compiler.target>

 4. Создаем класс: 

	@Mojo(name = "version")
	public final class VersionMojo extends AbstractMojo {
	
	    @Parameter(defaultValue = "${project}", required = true, readonly = true)
	    public MavenProject project;
	
	    public void execute() {
	        getLog().info("Project version: " + project.getVersion());
	    }
	}

	Чтобы указать, что созданный класс — это Mojo, указываем ему аннотацию @Mojo(name = “version”), где version — наименование цели. 
	Также наследуемся от класса AbstractMojo, который имплементирует всю необходимую для Mojo функциональность. 
	Нам нужно только реализовать метод execute. 
	В качестве параметра укажем текущий Maven проект, и в методе execute будем логировать версию проекта, т.е. нашего плагина.

 5. Добавляем необходимые зависимости с сайта https://mvnrepository.com

	    <dependencies>
	        <!-- https://mvnrepository.com/artifact/org.apache.maven/maven-plugin-api -->
	        <dependency>
	            <groupId>org.apache.maven</groupId>
	            <artifactId>maven-plugin-api</artifactId>
	            <version>3.8.5</version>
	        </dependency>
	        <!-- https://mvnrepository.com/artifact/org.apache.maven.plugin-tools/maven-plugin-annotations -->
	        <dependency>
	            <groupId>org.apache.maven.plugin-tools</groupId>
	            <artifactId>maven-plugin-annotations</artifactId>
	            <version>3.6.4</version>
	            <scope>provided</scope>
	        </dependency>
	        <!-- https://mvnrepository.com/artifact/org.apache.maven/maven-project -->
	        <dependency>
	            <groupId>org.apache.maven</groupId>
	            <artifactId>maven-project</artifactId>
	            <version>2.2.1</version>
	        </dependency>
	    </dependencies>

	Зависимость maven-plugin-api содержит необходимые классы и интерфейсы для создания Mojo (например, AbstractMojo).
	Зависимость maven-plugin-annotations содержит аннотации (@Mojo, @Parameter).
	Зависимость maven-project предоставит нам доступ к информации о проекте.

 6. Указываем в pom, что создаем именно плагин, для этого добавляем строку <packaging>maven-plugin</packaging> в конце блока groupId, artifactId, version

	<packaging>maven-plugin</packaging>

 7. Собираем и запускаем цель плагина:

	mvn clean install (mvn clean install -e - для того чтобы увидеть стек ошибок)
	или на вкладке Maven в Idea выбираем clean и install и нажимаем пуск
